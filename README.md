# Реферальная программа «Рабочие Руки»

Репозиторий с материалами по проекту реферальной программы для платформы «Рабочие Руки»: техническое задание, кликабельные прототипы интерфейса (мобильное приложение исполнителя + CRM маркетолога) и аналитика расхождений по условиям программы.

## О проекте

Реферальная программа позволяет действующим исполнителям платформы приглашать новых по персональной ссылке и получать вознаграждение за их активность. В составе:

- **Мобильное приложение исполнителя** — Личный кабинет с кнопкой «Реферальная программа»; экран программы с ссылкой, hero-суммой, числом приведённых и списком рефералов (ФИО замаскированы); кнопка `i` (как работает) и «Задать вопрос» (Telegram / MAX / Email / телефон).
- **CRM маркетолога** — 5 вкладок: Дашборд (CAC, активные рефереры, ROAS, воронка), Реф-связи (1 254 пары приглашающий↔реферал с фильтрами), Ссылки (личные ссылки исполнителей + ROAS), Финансы (выплаты, заморозка, Cap-защита, реестр), Настройки (коэффициенты, контакты поддержки, milestone-ставки, антифрод, hero-сумма).
- **Логика откатов** — окно мягкого удаления 30 дней, ручные корректировки счётчика заданий с журналом и правами (маркетолог / операционка / поддержка), списание с баланса при отмене после выплаты milestone'а.

Прототипы кликабельные, реализованы единым HTML-файлом без сборки и зависимостей — открываются в браузере двойным кликом.

## Как открыть прототип на локальной машине

### Способ 1 — двойной клик (самый простой)

1. Склонируйте репозиторий или скачайте ZIP:
   ```bash
   git clone https://github.com/azamdoctor/REF_RR.git
   cd REF_RR
   ```
2. Откройте `prototype_v6.html` двойным кликом — он запустится в системном браузере. Никакая сборка и сервер не нужны.

### Способ 2 — через локальный HTTP-сервер (рекомендуется)

Некоторые браузеры из соображений безопасности блокируют часть функциональности при открытии HTML через `file://`. Чтобы избежать сюрпризов, поднимите локальный сервер.

**Через Python (есть на macOS и Linux из коробки):**
```bash
cd REF_RR
python3 -m http.server 8000
```
Затем откройте: <http://localhost:8000/prototype_v6.html>

**Через Node.js (если установлен):**
```bash
cd REF_RR
npx serve .
```
Адрес покажется в терминале (обычно <http://localhost:3000>).

**Через VS Code:**
Установите расширение **Live Server**, откройте папку, кликните правой кнопкой на `prototype_v6.html` → *Open with Live Server*.

### Остановить сервер
В терминале, где запущен сервер, нажмите `Ctrl+C`.

## Навигация по прототипу v6

После открытия `prototype_v6.html`:

- **Сверху** — переключатель «Мобильное приложение» / «CRM».
- **Мобильное приложение** — 2 экрана подряд:
  - **Личный кабинет** с подсвеченной кнопкой «Реферальная программа».
  - **Экран программы** — ссылка, hero-сумма, стат-блок (приведено / сумма выплат), список рефералов с замаскированными ФИ, кнопки `i` и «Задать вопрос».
- **CRM** — 5 вкладок:
  - **Дашборд** — CAC, активные рефереры, ROAS, воронка «клик → 10 заданий», график новых рефералов, топ-10 приглашающих.
  - **Реф-связи** — таблица пар приглашающий↔реферал (ФИО+телефон), фильтры, антифрод-флаги, кнопка «Открыть» → карточка реферала.
  - **Ссылки** — KPI по личным ссылкам, таблица 186 активных ссылок, конверсии и ROAS.
  - **Финансы** — выплачено за месяц, в заморозке, расходы по месяцам, Cap-защита, реестр выплат.
  - **Настройки** — расчётные коэффициенты (включая редактируемую hero-сумму), контакты поддержки, milestone-ставки, шаблон сообщения при копировании, антифрод-пороги. Каждая панель — со своей кнопкой «Сохранить», которая появляется только при изменении полей.

## Открытые вопросы по проекту

- Расхождение в ставках вознаграждения между источниками (docx Ирины: 100/500/3000 vs Bitrix #67031: 1500/5500).

- Реализация автоматизации через ИИ-агента (упомянуто в #53413 Александром Мартыненко).
